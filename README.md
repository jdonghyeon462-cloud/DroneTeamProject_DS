# DroneProject

## 프로젝트 개요

DroneProject는 C언어로 구현한 자폭 드론 시뮬레이터 프로젝트입니다. Visual Studio 2019 환경에서 작성되었으며, `DroneProject.c`의 `main` 함수가 전체 실행 흐름을 제어합니다.

프로그램은 좌표 생성, 좌표 정렬, 구간별 거리 및 배터리 소모량 계산, 배터리 한계 초과 시 안전 회귀 경로 출력 순서로 동작합니다.

## 개발 환경

| 항목 | 내용 |
|---|---|
| 언어 | C |
| 개발 도구 | Visual Studio 2019 |
| 실행 방식 | Console Application |
| 주요 개념 | 파일 입출력, 구조체, 동적 할당, 연결 리스트, 스택 |

## 소스 파일 설명

| 파일명 | 역할 |
|---|---|
| `DroneProject.c` | `main` 함수가 있는 메인 파일이며, `run01`, `run02`, `run03`, `run04` 함수를 순서대로 호출합니다. |
| `01.c` | 난수를 이용해 드론 목적지 좌표를 생성하고 `01.txt`에 저장합니다. |
| `02.c` | `01.txt`의 좌표를 읽고 원점 기준 거리순으로 연결 리스트에 정렬한 뒤 `02.txt`에 저장합니다. |
| `03.c` | `02.txt`의 정렬된 좌표를 기반으로 구간별 거리, 배터리 상수, 배터리 소모량을 계산하고 `03.txt`에 저장합니다. |
| `04.c` | `03.txt`의 구간 데이터를 읽어 누적 배터리 소모량을 계산하고, 배터리 한계 초과 시 스택을 이용해 안전 회귀 경로를 출력합니다. |
| `cdh.h` | 공통 헤더 파일입니다. 구조체 정의, 상수 정의, 함수 원형 선언을 포함합니다. |

## 실행 흐름

```text
DroneProject.c
    ↓
run01()
    ↓
01.txt 생성
    ↓
run02()
    ↓
02.txt 생성
    ↓
run03()
    ↓
03.txt 생성
    ↓
run04()
    ↓
04.txt 생성
```

## 주요 기능

1. 드론 좌표 생성
2. 좌표 중복 방지
3. 원점 기준 거리 계산
4. 연결 리스트를 이용한 거리순 정렬
5. 구간별 배터리 소모량 계산
6. 스택을 이용한 안전 회귀 경로 추적
7. 파일 입출력을 이용한 단계별 데이터 저장

## 실행 방법

1. Visual Studio 2019에서 프로젝트를 엽니다.
2. `DroneProject.c` 파일에만 `main` 함수가 있는지 확인합니다.
3. `01.c`, `02.c`, `03.c`, `04.c`, `cdh.h`가 같은 프로젝트에 포함되어 있는지 확인합니다.
4. `Ctrl + F5`를 눌러 실행합니다.
5. 실행 중 열리는 `01.txt`, `02.txt`, `03.txt`, `04.txt` 파일은 확인 후 닫아야 다음 단계로 진행됩니다.

## 생성되는 파일

| 파일명 | 설명 |
|---|---|
| `01.txt` | 생성된 좌표 데이터 |
| `02.txt` | 거리순으로 정렬된 좌표 데이터 |
| `03.txt` | 구간별 거리와 배터리 소모량 데이터 |
| `04.txt` | 최종 임무 수행 결과 및 안전 회귀 경로 데이터 |

## 코드 구조 특징

이 프로젝트는 여러 개의 C 소스 파일을 하나의 실행 파일로 빌드하는 구조입니다. 각 파일은 독립적인 `main` 함수를 가지지 않고, `run01`, `run02`, `run03`, `run04` 함수 형태로 기능을 분리합니다.

`DroneProject.c`의 `main` 함수는 각 단계를 순서대로 실행하며, 이전 단계에서 생성된 텍스트 파일을 다음 단계의 입력 파일로 사용합니다.

## 주의 사항

`01.c`, `02.c`, `03.c`, `04.c` 파일에는 `main` 함수가 없어야 합니다. `main` 함수는 반드시 `DroneProject.c`에만 존재해야 합니다.

Visual Studio에서 실행할 때 모든 `.c` 파일이 프로젝트에 포함되어 있어야 합니다. 특정 파일이 빌드에서 제외되어 있으면 외부 참조 오류가 발생할 수 있습니다.
